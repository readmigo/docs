## 概述

本文档评估创建一个类似 Project Gutenberg 的中文公共领域电子书平台的可行性，综合分析法律、内容、技术、市场和商业等多个维度。

---

### 市场痛点

| 痛点 | 当前状况 | 机会 |
|------|----------|------|
| 内容分散 | 古籍资源分布在多个平台，无统一入口 | 一站式聚合 |
| 格式不统一 | PDF/图片为主，阅读体验差 | 标准化 EPUB |
| 质量参差 | 缺乏统一校对标准 | 专业校对体系 |
| 商业门槛 | 部分平台需付费或注册 | 完全免费开放 |
| 移动体验 | 多数平台移动端适配差 | 原生 App 支持 |

---

### 进入公共领域的作品分类

| 类别 | 时间范围 | 代表作品/作者 | 可用性 |
|------|----------|---------------|--------|
| **古典文学** | 1900年前 | 四大名著、二十四史、诸子百家 | ✅ 完全可用 |
| **清末民初** | 1900-1925 | 梁启超、章太炎、王国维 | ✅ 完全可用 |
| **民国作家** | 1925-1949 | 鲁迅(1936卒)、茅盾(1981卒) | ⚠️ 部分可用 |
| **当代作家** | 1949后 | 巴金(2005卒)、冰心(1999卒) | ❌ 暂不可用 |

### 法律风险评估

| 风险类型 | 等级 | 说明 | 应对措施 |
|----------|------|------|----------|
| 古籍版权争议 | 低 | 古籍原文无版权，但校注本可能有 | 使用通行本 |
| 作家遗产纠纷 | 中 | 继承人可能主张权益 | 严格核实去世时间 |
| 跨境法律冲突 | 中 | 不同国家保护期限不同 | 按最严格标准执行 |
| 编辑版权 | 低 | 整理、校注可能产生新版权 | 自主整理或使用公版 |

#### CText (中国哲学书电子化计划)

**优势**:
- 30,000+ 标题、50亿+ 字数
- 专业学术校对，质量极高
- 先秦两汉文献最权威
- 支持 API 访问

**劣势**:
- API 需要认证申请
- 商业使用需要单独授权
- 专注哲学经典，覆盖面有限

#### 书格 (Shuge.org)

**优势**:
- 高清扫描件，保留原貌
- CC BY 4.0 许可，商用友好
- 社区活跃，持续更新

**劣势**:
- 以 PDF 扫描为主，非结构化文本
- 无 API 接口
- 不适合移动阅读

#### 识典古籍 (北大-字节跳动)

**优势**:
- OCR + AI 辅助，文本化程度高
- 现代化界面和搜索
- 国家级资源支持

**劣势**:
- 商业使用条款不明确
- 无公开 API
- 平台依赖性强

#### 维基文库中文

**优势**:
- 完全开放，CC BY-SA 许可
- MediaWiki API 免费使用
- 社区校对机制

**劣势**:
- 内容相对较少
- 校对质量不均
- 近现代文学版权争议

### 与英文 Gutenberg 对比

| 维度 | Project Gutenberg (英文) | 中文公版书 |
|------|--------------------------|------------|
| 核心作品数 | 76,000+ | 3,000-5,000 |
| 年增长量 | 500+/年 | 50-100/年 |
| 内容深度 | 广而全 | 经典精选 |
| 电子化程度 | 高 (EPUB/TXT) | 中 (多为 PDF) |
| 元数据完整度 | 高 | 低 |
| 社区贡献 | 活跃 | 相对较少 |

### 技术挑战

| 挑战 | 难度 | 解决方案 |
|------|------|----------|
| 古籍 OCR | ⭐⭐⭐⭐ | 使用专业古籍 OCR 引擎 (识典/百度) |
| 繁简转换 | ⭐⭐ | 开源工具 (OpenCC) + 人工校对 |
| 异体字处理 | ⭐⭐⭐ | Unicode 标准化 + 异体字表 |
| 古文断句 | ⭐⭐⭐⭐ | AI 模型辅助 + 专家审核 |
| 元数据整理 | ⭐⭐⭐ | 参考权威目录 (四库/中图分类) |
| 版本校勘 | ⭐⭐⭐⭐⭐ | 选用通行善本，标注版本来源 |

### 现有竞品

| 竞品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **微信读书** | 综合阅读 | 用户基数大、社交功能 | 公版书非核心、需会员 |
| **藏书馆** | 电子借阅 | 图书馆资源整合 | 借阅限制、非公版定位 |
| **古诗文网** | 诗词工具 | 诗词专精、注释丰富 | 仅限诗词、无完整书籍 |
| **书格** | 古籍影印 | 高清原貌、学术价值 | PDF 格式、阅读体验差 |
| **殆知阁** | 古籍收录 | 资源丰富 | 界面老旧、无移动端 |

### 可选商业模式

| 模式 | 说明 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **纯公益** | 捐赠支持 | 无利益冲突 | 资金不稳定 | ⭐⭐ |
| **Freemium** | 基础免费+增值服务 | 可持续发展 | 需平衡体验 | ⭐⭐⭐⭐⭐ |
| **广告模式** | 免费+广告 | 门槛低 | 影响阅读 | ⭐⭐ |
| **企业授权** | To B 服务 | 客单价高 | 市场有限 | ⭐⭐⭐ |
| **教育合作** | 校园渠道 | 用户精准 | 决策周期长 | ⭐⭐⭐⭐ |

### 风险矩阵

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 版权纠纷 | 低 | 高 | 严格版权审核流程 |
| 资金不足 | 中 | 高 | 多元化收入来源 |
| 技术难题 | 中 | 中 | 渐进式开发、借力开源 |
| 用户增长慢 | 高 | 中 | 差异化定位、精准营销 |
| 大厂竞争 | 中 | 中 | 专注细分、社区驱动 |
| 政策变化 | 低 | 高 | 合规运营、分布式存储 |

### 多维度评分

| 评估维度 | 评分 | 说明 |
|----------|------|------|
| **法律可行性** | ⭐⭐⭐⭐ | 古籍无障碍，近现代需谨慎 |
| **内容可用性** | ⭐⭐⭐ | 核心经典充足，总量有限 |
| **技术可行性** | ⭐⭐⭐⭐ | 成熟方案可用，古籍处理有难度 |
| **市场需求** | ⭐⭐⭐⭐ | 需求真实存在，市场待培育 |
| **商业可行性** | ⭐⭐⭐ | Freemium 可行，纯公益困难 |
| **竞争环境** | ⭐⭐⭐ | 有差异化空间，但巨头在场 |
| **资源投入** | ⭐⭐⭐ | 需要持续投入，但可控 |

---

### 关键成功因素

1. **差异化定位**: 专注阅读体验，而非单纯内容堆砌
2. **质量优先**: 宁精勿滥，建立口碑
3. **社区驱动**: 借助志愿者力量，降低运营成本
4. **开放生态**: API 开放，吸引开发者
5. **合规运营**: 严格版权审核，避免法律风险

# Operations 模块功能指南

## 概述

Operations 模块是 Readmigo 运营管理平台的核心组件，提供全面的数据分析和运营决策支持。本文档详细介绍各子模块的功能、指标定义、数据解读方法和行动建议。

---

## 1. Operations Overview（运营概览）

### 业务价值

提供 Readmigo 平台整体运营健康状况的实时快照，帮助管理层快速了解核心业务指标。

### 核心指标

#### 用户活跃度指标

| 指标 | 说明 | 计算方法 |
|------|------|---------|
| DAU (日活跃用户) | 当日活跃的总用户数 | 按 iOS/Android/Web 分平台统计 |
| MAU (月活跃用户) | 最近 30 天活跃的用户数 | 评估用户池规模 |
| 环比变化 | DAU 与昨日对比 | 变化量和百分比 |

#### 用户增长指标

| 指标 | 说明 | 平台分类 |
|------|------|---------|
| 新增用户 | 当日新注册用户数 | iOS / Android / Web |
| 新增变化 | 与昨日新增用户数的对比 | 绝对值和环比 |

#### 阅读行为指标

| 指标 | 说明 | 业务意义 |
|------|------|---------|
| 总阅读时长 | 当日所有用户累计阅读分钟数 | 衡量内容消费总量 |
| 总阅读页数 | 当日所有用户累计阅读页数 | 阅读深度指标 |
| 完成书籍数 | 当日完成阅读的书籍总数 | 内容完成度 |
| 阅读会话数 | 当日发起的阅读会话总数 | 用户打开频次 |
| 人均阅读时长 | 平均每个活跃用户的阅读时长 | 用户粘性核心指标 |

#### AI 交互指标

| 指标 | 说明 | 成本控制 |
|------|------|---------|
| AI 交互次数 | AI 对话、翻译、解释等功能的使用次数 | 功能使用率 |
| Token 消耗 | AI 服务消耗的 Token 总量 | 技术成本监控 |
| AI 成本 (USD) | AI 服务产生的费用 | 财务成本监控 |

#### 内容互动指标

| 指标 | 说明 |
|------|------|
| 分享次数 | 用户分享内容的总次数 |
| 明信片生成 | 用户生成的明信片数量 |
| 划线标注 | 用户创建的高亮/划线数量 |
| 广场发帖 | 用户在社区广场的发帖数 |
| 单词学习/复习 | 单词本功能的使用数据 |

### 可视化组件

- **指标卡片**：DAU、MAU、新增用户、总阅读时长等关键指标，带环比变化箭头（↑ 绿色 / ↓ 红色）
- **趋势图表**：7 天/30 天的 DAU、阅读时长、AI 交互趋势折线图
- **热门书籍列表**：当日最受欢迎的 Top 5 书籍，带封面、标题、作者、阅读数据

### 数据解读

| 情况 | 可能原因 | 行动建议 |
|------|---------|---------|
| DAU 环比增长 > 5% | 新功能上线、营销活动、热点事件 | 持续观察，复制成功经验 |
| DAU 环比下降 > 10% | 技术故障、体验问题、竞品冲击 | 立即排查技术日志和用户反馈 |
| 人均阅读时长 < 15 分钟 | 内容推荐不精准、阅读体验差 | 优化推荐算法，改进阅读器 |
| AI 成本快速增长（周增长 > 20%） | AI 功能滥用、成本效率低 | 评估 AI 功能使用效率，考虑限流或优化 Prompt |
| 新增用户稳定但 DAU 下降 | 留存率出现问题 | 重点关注留存分析模块 |

### 使用场景

**每日例会检查清单：**

1. ✅ DAU 是否正常（与昨日对比）
2. ✅ 新增用户是否达标（与周平均对比）
3. ✅ 人均阅读时长是否健康（> 20 分钟）
4. ✅ AI 成本是否超预算
5. ✅ 热门书籍是否有异常（如某书突然爆火）

## 2. Reading Stats（阅读统计）

### 业务价值

深入分析用户阅读行为模式，识别受欢迎的内容和用户偏好，为内容运营和推荐算法提供数据支持。

### 核心指标

#### 总体概览指标

| 指标 | 说明 | 业务意义 |
|------|------|---------|
| 总阅读时长 | 指定时间段内累计阅读分钟数 | 内容消费总量 |
| 总阅读会话数 | 用户发起的阅读会话总数 | 打开频次 |
| 活跃读者数 | 有阅读行为的独立用户数 | 内容触达率 |
| 在读书籍数 | 用户正在阅读的书籍总数 | 内容丰富度 |
| 平均会话时长 | 单次阅读会话的平均时长 | 用户专注度 |
| 人均每日阅读时长 | 平均每个用户每天的阅读时长 | 用户粘性 |

#### 书籍排行指标

| 指标 | 说明 | 排序维度 |
|------|------|---------|
| 书籍阅读时长 | 每本书累计被阅读的总时长 | 受欢迎程度 |
| 独立读者数 | 阅读该书的独立用户数 | 覆盖广度 |
| 阅读会话数 | 该书的阅读会话总数 | 打开频次 |
| 人均阅读时长 | 平均每个读者在该书上的时长 | 内容吸引力 |

#### 用户排行指标

| 指标 | 说明 | 用户画像 |
|------|------|---------|
| 用户阅读时长 | 单个用户的累计阅读时长 | 活跃度评估 |
| 阅读书籍数 | 用户阅读的不同书籍数量 | 兴趣广度 |
| 阅读会话数 | 用户发起的阅读会话数 | 使用频率 |
| 完成书籍数 | 用户读完的书籍总数 | 完成度评估 |

#### 分类统计指标

| 指标 | 说明 | 内容策略 |
|------|------|---------|
| 分类阅读时长 | 每个书籍分类的累计阅读时长 | 内容偏好 |
| 分类独立读者 | 阅读该分类的独立用户数 | 受众规模 |
| 分类占比 | 该分类在总阅读时长中的占比 | 内容配比 |

#### 时间模式指标

| 指标 | 说明 | 运营价值 |
|------|------|---------|
| 时段分布 | 0-23 点每个小时的阅读活跃度 | 推送时机优化 |
| 星期分布 | 周一到周日的阅读活跃度对比 | 活动排期 |

### 数据来源

```
reading_sessions (阅读会话记录)
    ├── user_id          → users (用户信息)
    ├── user_book_id     → user_books (用户-书籍关系)
    │                        └── book_id → books (书籍元数据)
    ├── started_at       (会话开始时间)
    ├── ended_at         (会话结束时间)
    └── duration_minutes (阅读时长)

daily_stats (用户每日统计)
    ├── user_id
    ├── date
    ├── reading_minutes
    ├── pages_read
    └── books_finished
```

### 可视化组件

- **概览卡片**：总阅读时长、活跃读者数等关键指标，带图标和趋势箭头
- **书籍排行榜**：按阅读时长或读者数排序的书籍列表，包含封面、标题、作者、统计数据
- **用户排行榜**：最活跃读者列表，显示用户名、阅读时长、书籍数、完成数
- **分类饼图**：各书籍分类的阅读时长占比，带颜色编码
- **时段热力图**：24 小时阅读活跃度分布，颜色深浅表示活跃程度
- **星期柱状图**：周一到周日的阅读时长对比
- **趋势折线图**：阅读时长、活跃读者的时间变化（可选 7 天/30 天/90 天）

### 数据解读

| 情况 | 可能原因 | 行动建议 |
|------|---------|---------|
| 高阅读时长但低读者数的书籍 | 深度爱好者专属，小众精品 | 精准推荐给相似兴趣用户 |
| 高读者数但低人均时长的书籍 | 标题党、内容吸引力不足 | 考虑下架或替换更优质版本 |
| 晚上 8-10 点阅读高峰 | 用户睡前阅读习惯 | 在此时段发送推送通知 |
| 周末阅读时长下降 | 用户外出活动更多 | 推出离线阅读功能、户外场景优化 |
| 某分类阅读时长激增 | 热点事件相关（如电影上映） | 及时补充该分类书籍，首页推荐 |
| 用户阅读书籍数多但完成率低 | 内容质量参差不齐、推荐不精准 | 优化推荐算法，提升内容质量 |

### 使用场景

**周会内容运营复盘：**

1. ✅ 查看 Top 10 热门书籍，评估推荐策略效果
2. ✅ 识别冷门但高质量书籍（高人均时长），定向推送
3. ✅ 分析分类占比变化，调整内容采购计划
4. ✅ 根据时段分布优化推送通知时间
5. ✅ 识别异常阅读行为用户，提供个性化关怀

## 3. Retention Analysis（留存分析）

### 业务价值

留存率是衡量产品健康度的核心指标。通过留存分析，识别用户流失的关键节点，评估产品改进和运营活动的效果。

### 核心指标

#### 留存率概览指标

| 指标 | 定义 | 计算公式 | 健康标准 |
|------|------|---------|---------|
| D1 留存率（次日留存） | 用户注册后第 2 天仍活跃的比例 | (D1 活跃用户数 / 队列用户数) × 100% | > 40% |
| D7 留存率（7 日留存） | 用户注册后第 8 天仍活跃的比例 | (D7 活跃用户数 / 队列用户数) × 100% | > 25% |
| D30 留存率（30 日留存） | 用户注册后第 31 天仍活跃的比例 | (D30 活跃用户数 / 队列用户数) × 100% | > 15% |
| 环比变化 | 与上一周期的留存率对比 | (本期留存率 - 上期留存率) / 上期留存率 | 正增长 |

#### 队列分析指标

| 指标 | 说明 | 分析价值 |
|------|------|---------|
| 队列留存矩阵 | 按注册日期分组的用户留存热力图 | 识别留存模式和异常队列 |
| 队列规模 | 每个队列的用户数量 | 评估获客效果 |
| 队列留存轨迹 | 单个队列在 D1/D7/D30 的留存变化 | 流失节点分析 |

#### 渠道对比指标

| 指标 | 说明 | 决策支持 |
|------|------|---------|
| 渠道留存率 | 不同获客渠道的留存率对比 | 渠道质量评估 |
| 渠道用户数 | 各渠道贡献的用户数量 | 渠道规模评估 |
| 渠道 ROI | 结合成本数据评估投资回报 | 预算分配优化 |

### 留存定义

```
用户注册时间轴示例：

注册日        D1         D7                                D30
  │           │          │                                  │
  ▼           ▼          ▼                                  ▼
Day 0    Day 1~2    Day 7~8                           Day 30~31
  │           │          │                                  │
  └───────────┴──────────┴──────────────────────────────────┘
              │          │                                  │
         检查是否活跃  检查是否活跃                     检查是否活跃

活跃定义：在检查窗口期内有以下任一行为
- 打开应用（更新 lastActiveAt）
- 发起阅读会话
- AI 交互
- 社区发帖/评论
```

### 数据来源

```
user_retention (预计算的队列留存数据)
    ├── cohort_date          (队列日期 = 用户注册日期)
    ├── channel              (注册渠道：organic, facebook, google 等)
    ├── cohort_size          (该队列的用户数)
    ├── day1_retained        (D1 留存用户数)
    ├── day7_retained        (D7 留存用户数)
    ├── day30_retained       (D30 留存用户数)
    ├── day1_retention_rate  (D1 留存率 = day1_retained / cohort_size)
    ├── day7_retention_rate  (D7 留存率)
    ├── day30_retention_rate (D30 留存率)
    └── calculated_at        (计算时间)

Cron Job 调度：
- 每日凌晨 2:00 运行
- 计算前一天注册用户的 D1 留存
- 计算 7 天前注册用户的 D7 留存
- 计算 30 天前注册用户的 D30 留存
```

### 可视化组件

#### 概览卡片

显示 D1/D7/D30 留存率，带环比变化指示器：

```
┌─────────────────────────────────────────────────────┐
│  D1 留存率: 45.3%  ↑ 2.5%                           │
│  D7 留存率: 28.7%  ↑ 1.2%                           │
│  D30 留存率: 16.8%  ↓ 0.8%                          │
└─────────────────────────────────────────────────────┘

↑ 绿色 = 环比增长
↓ 红色 = 环比下降
```

#### 队列热力图

横轴为注册日期，纵轴为留存天数，颜色深浅表示留存率：

```
        Day 1    Day 7    Day 30

2026-01-02  ███      ██       █      (60-80% 黄绿)
2026-01-03  ██       █        -      (40-60% 黄色)
2026-01-04  █        -        -      (20-40% 橙色)
2026-01-05  -        -        -      (0-20%  红色)
```

#### 渠道对比柱状图

不同渠道的 D1/D7/D30 留存率并列对比，快速识别高质量渠道。

#### 趋势折线图

选择 D1/D7/D30 指标的时间序列变化，识别趋势和异常点。

### 数据解读

| 情况 | 诊断 | 行动建议 |
|------|------|---------|
| D1 留存率 < 40% | 新手引导有问题，首次体验不佳 | 优化注册流程、新手引导、首页推荐 |
| D7 留存率大幅下降（D1 60% → D7 20%） | 缺乏长期价值，内容推荐不够吸引 | 增加推送提醒、个性化推荐、签到奖励 |
| D30 留存率稳定 | 产品形成习惯，用户粘性强 | 引入付费会员、社区功能强化粘性 |
| 某队列留存率异常高 | 该批用户受特定活动吸引 | 分析用户特征（来源、首次阅读书籍）并复制 |
| Organic 渠道留存率 > 付费渠道 | 自然流量质量更高 | 减少付费投放，强化 SEO 和口碑传播 |
| Facebook 渠道留存率极低（< 20%） | 广告定向或落地页有问题 | 停止该渠道投放，优化广告创意和定向 |

### 使用场景

**周会留存复盘流程：**

1. ✅ 查看 D1/D7/D30 留存率趋势，识别异常波动
2. ✅ 分析队列热力图，找出高留存和低留存队列
3. ✅ 对比不同渠道留存率，调整市场预算分配
4. ✅ 结合产品迭代日志，评估功能上线对留存的影响
5. ✅ 制定针对性留存提升方案（如新手引导优化、推送策略调整）

## 4. User Demographics（用户画像统计）

### 业务价值

了解用户的人口统计学特征，为产品设计、内容采购、市场推广、商业化策略提供数据依据。

### 核心指标

#### 总体概览指标

| 指标 | 说明 | 健康标准 |
|------|------|---------|
| 用户总数 | 平台注册用户总数 | 持续增长 |
| 完善资料用户数 | 填写了个人资料的用户数 | - |
| 资料完善率 | 完善资料用户占比 | > 40% |
| 各性别用户数 | Male/Female/Other/Prefer not to say/Unknown | 相对平衡 |
| 平均年龄 | 所有填写年龄用户的平均年龄 | 符合目标用户群 |
| 主要国家 | Top 3 用户来源国家 | 地理分布合理 |

#### 性别分布指标

| 指标 | 说明 | 分析维度 |
|------|------|---------|
| 性别占比 | 各性别用户的百分比 | 用户结构 |
| 性别活跃度 | 各性别用户的平均阅读时长、会话数 | 行为差异 |
| 性别内容偏好 | 不同性别用户喜欢的书籍分类 | 内容策略 |

#### 年龄分布指标

| 年龄段 | 说明 | 特征 |
|--------|------|------|
| Under 18 | 未成年用户 | 学生群体 |
| 18-24 | 大学生、职场新人 | 高活跃度、低付费能力 |
| 25-34 | 职场主力、家庭初建 | 中高活跃度、中付费能力 |
| 35-44 | 职场中坚、家庭稳定 | 中活跃度、高付费能力 |
| 45-54 | 职场高层、子女成长 | 低活跃度、高付费能力 |
| 55+ | 退休群体 | 低活跃度、时间充裕 |

#### 地理分布指标

| 指标 | 说明 | 分析粒度 |
|------|------|---------|
| 国家分布 | 用户所在国家 Top 20 | 国际化策略 |
| 省份分布 | 用户所在省份 Top 20（中国） | 地域运营 |
| 城市分布 | 用户所在城市 Top 20 | 本地化活动 |
| 地理活跃度 | 各地区用户的平均阅读时长 | 用户质量评估 |

### 数据来源

```
users (用户基本信息)
    ├── id
    ├── email
    ├── created_at       (注册时间)
    └── last_active_at   (最后活跃时间)

user_profiles (用户详细资料)
    ├── user_id
    ├── gender           (性别：MALE, FEMALE, OTHER, PREFER_NOT_TO_SAY, UNKNOWN)
    ├── birth_year       (出生年份，用于计算年龄)
    ├── country          (国家)
    ├── province         (省份)
    └── city             (城市)

reading_sessions (用户活跃行为数据)
    ├── user_id
    ├── started_at
    └── duration_minutes
```

### 可视化组件

#### 性别饼图

各性别占比的圆形图，带颜色编码：

```
┌───────────────────────────────────────────┐
```

## 附录：数据驱动决策案例

### 案例 1：发现 D7 留存率从 45% 下降到 30%

**分析：**
- 检查队列热力图，发现最近 3 天注册的用户 D7 留存率异常低
- 对比产品迭代日志，发现新版本上线时间与留存下降吻合
- 用户反馈显示新手引导流程有 bug，导致部分用户无法完成首次阅读

**行动：**
1. 紧急修复新手引导 bug，并发布 hotfix
2. 设计"7 日阅读挑战"活动，增加 Push 通知频率
3. 优化推荐算法，确保新用户前 7 天每天都能收到感兴趣的书籍推荐
4. 监控后续队列留存率，验证修复效果

**结果：**
- 修复后 3 天，D7 留存率回升至 42%
- 阅读挑战活动使 D7 活跃用户增加 15%

---

### 案例 2：某小众分类阅读时长激增 200%

**分析：**
- Reading Stats 显示"科幻小说"分类阅读时长从日均 2000 分钟激增至 6000 分钟
- 查看热门书籍榜单，发现《三体》阅读数据暴涨
- 外部调研显示《三体》电视剧最近热播，引发阅读热潮

**行动：**
1. 紧急采购更多科幻小说（如刘慈欣其他作品、阿西莫夫系列）
2. 首页增加"科幻热潮"专题推荐位
3. 在社交媒体推广科幻书籍，吸引更多用户
4. 设计"科幻阅读打卡"活动，延长热度周期

**结果：**
- 科幻分类 MAU 增长 80%
- 新增用户中 30% 首次阅读科幻书籍
- 专题推荐点击率达到 15%，远超平均水平

---

### 案例 3：Facebook 渠道留存率仅 15%，远低于 Organic 的 50%

**分析：**
- Retention Analysis 显示 Facebook 渠道 D1 留存率 25%，D7 留存率 15%，D30 留存率 8%
- Organic 渠道 D1 留存率 60%，D7 留存率 50%，D30 留存率 40%
- 用户画像分析显示 Facebook 渠道用户平均年龄 18-24 岁，而 Organic 渠道平均年龄 25-34 岁
- Facebook 广告定向为"兴趣：阅读"，但实际吸引了大量"广告点击习惯"用户

**行动：**
1. 停止 Facebook 广告投放，节省预算
2. 将预算转移到 Google Search Ads（搜索意图更精准）
3. 优化广告创意，强调"深度阅读"而非"免费书籍"
4. 测试 Instagram Ads 定向为"关注读书博主"的用户

**结果：**
- Google Search Ads 渠道 D1 留存率达到 55%
- CPA（单个用户获取成本）从 $5 降至 $3
- 整体获客 ROI 提升 40%

```
✅ Operations Overview
   - DAU 是否正常（与昨日对比，波动 < 10%）
   - 新增用户是否达标（与周平均对比）

   - API 响应时间（P95 < 1s）
   - 5xx 错误率（< 0.1%）

   - 是否有批量相似投诉
   - 是否有严重 bug 报告
```

```
✅ Reading Stats 深度分析
   - 上周 Top 10 热门书籍，评估推荐策略效果

   - 根据时段分布优化推送通知时间

✅ Retention Analysis
   - 查看 D1/D7/D30 留存率趋势，识别异常波动
   - 分析队列热力图，找出高留存和低留存队列
   - 对比不同渠道留存率，调整市场预算分配

✅ 制定本周运营计划
   - 确定本周内容推荐主题

```

```
   - 内容采购计划（基于分类阅读数据）
   - 产品优化计划（基于留存分析）
```

```
🚨 Performance Dashboard 发现故障

   → 评估推荐算法是否有 bug
   → 及时下架问题内容或调整推荐权重

🚨 AI 成本单日激增 > 50%
   → 排查是否有 AI 功能滥用或攻击
   → 检查是否有代码 bug 导致重复调用
   → 临时启用限流措施，保护成本
```

## 总结

Operations 模块是 Readmigo 数据驱动决策的核心工具，涵盖了从用户增长、内容消费、用户留存到用户画像的全方位数据分析能力。通过合理使用这些模块，运营团队可以：

1. **实时监控**业务健康状况，及时发现问题
2. **深度分析**用户行为，优化产品和内容策略
3. **精准评估**渠道质量，优化市场预算分配
4. **数据驱动**产品迭代，提升用户体验和留存率
5. **科学决策**商业化策略，最大化用户生命周期价值

建议运营团队建立日常巡检机制，将数据分析融入每日例会、周会、月度复盘流程，形成"数据 → 洞察 → 行动 → 验证"的闭环，持续优化平台运营效果。

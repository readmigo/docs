# 中文公共领域书籍资源调研

> 作为 Readmigo 扩展中文书籍品类的资料储备

---

## 1. 调研背景

| 项目 | 描述 |
|------|------|
| **目标** | 寻找类似 Project Gutenberg / Standard Ebooks 的中文公共领域书籍来源 |
| **核心要求** | EPUB 格式优先、质量高、可批量获取 |
| **版权要求** | 公共领域或明确授权可商用 |

## 3. Tier 1 资源详细分析

### 3.1 好读 (Haodoo)

| 项目 | 描述 |
|------|------|
| **网站** | https://haodoo.net / https://haodoo.org |
| **书籍数量** | 3,300+ 本 |
| **格式** | EPUB、PDB（直式阅读） |
| **内容类型** | 世纪百强、武侠、言情、推理、奇幻、历史、有声书 |
| **质量特点** | 经站长及读友校对，错字较少 |
| **版权状态** | 公共领域作品为主 |

#### 核心优势

1. **直式排版** - 专为中文阅读优化，更接近实体书体验
2. **社区校对** - 2001年建站至今，读友持续贡献校对
3. **EPUB 支持** - 直接提供 EPUB 格式下载
4. **分类完善** - 武侠、言情、推理等通俗文学分类齐全

#### 内容分类

| 分类 | 书籍数量 | 适合人群 |
|------|----------|----------|
| 世纪百强 | 100+ | 文学爱好者 |
| 武侠小说 | 500+ | 武侠迷 |
| 推理小说 | 300+ | 悬疑爱好者 |
| 言情小说 | 400+ | 言情读者 |
| 奇幻小说 | 200+ | 奇幻爱好者 |
| 历史烟云 | 300+ | 历史爱好者 |
| 随身智囊 | 200+ | 国学入门 |

#### 数据获取方式

```
书籍列表: https://haodoo.net/?M=book&P=*
下载页面: https://haodoo.net/?M=d&P={book_id}
EPUB 下载: 解析下载页面获取直链
```

#### Readmigo 对接建议

| 阶段 | 任务 | 优先级 |
|------|------|--------|
| P0 | 抓取书籍列表和元数据 | 高 |
| P0 | 下载 EPUB 并解析 | 高 |
| P1 | 提取章节结构 | 中 |
| P1 | 文本难度分析 | 中 |
| P2 | 有声书关联 | 低 |

## 4. Tier 2 资源详细分析

### 4.1 国学大师

| 项目 | 描述 |
|------|------|
| **网站** | https://www.guoxuedashi.net |
| **书籍数量** | 20,000 种 / 28 万卷 / 约 24 亿字 |
| **格式** | 软件包 (6.5GB)、在线阅读 |
| **内容类型** | 古典文学全集 |
| **特点** | 中国常见古代图书最全的集成 |

#### 核心优势

1. **数量最全** - 囊括中国常见古代图书
2. **全文检索** - 软件支持全文搜索
3. **批量下载** - 6.5GB 软件包一次性获取
4. **影印古籍** - 8000 部古籍 PDF 可下载

#### 数据获取方式

```
软件下载: 网站提供国学大师软件安装包
影印古籍: https://www.guoxuedashi.net/guji/
在线阅读: https://www.guoxuedashi.net/a/
```

#### 对接难点

| 难点 | 解决方案 |
|------|----------|
| 非标准 EPUB | 需从软件包提取并转换格式 |
| 无 API | 需解析软件数据文件 |
| 古文为主 | 可能需要添加注释/翻译 |

### 4.2 维基文库中文版

| 项目 | 描述 |
|------|------|
| **网站** | https://zh.wikisource.org |
| **文档数量** | 290,000+ 篇 |
| **格式** | HTML、可导出多种格式 |
| **内容类型** | 文学、法律、历史文献 |
| **许可** | CC BY-SA |

#### 核心优势

1. **结构化数据** - MediaWiki API 可用
2. **持续更新** - 社区维护
3. **多格式导出** - 可导出 EPUB/PDF
4. **古籍全集** - 永乐大典、四库全书等

#### API 接口

```
基础 URL: https://zh.wikisource.org/w/api.php

获取页面内容:
GET /w/api.php?action=query&titles={title}&prop=revisions&rvprop=content&format=json

导出 EPUB (需使用 Wikisource 导出工具或第三方工具)
```

#### 对接建议

| 阶段 | 任务 |
|------|------|
| P1 | 筛选高质量完整作品 |
| P1 | 使用 API 批量获取内容 |
| P2 | 转换为标准 EPUB |
| P2 | 元数据标准化 |

## 6. 实施建议

### 6.1 分阶段实施路径

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Phase 1: 快速验证 (1-2 周)                                              │
│  ├── 对接好读 (Haodoo)，获取 500+ 本高质量 EPUB                          │
│  ├── 验证 EPUB 解析和章节提取流程                                        │
│  └── 难度分析算法适配中文                                                │
├─────────────────────────────────────────────────────────────────────────┤
│  Phase 2: 扩展内容 (2-4 周)                                              │
│  ├── 对接文硕阁，获取古典文学全集                                         │
│  ├── 分类体系映射                                                        │
│  └── 中文书籍推荐算法调优                                                │
├─────────────────────────────────────────────────────────────────────────┤
│  Phase 3: 深度补充 (持续)                                                │
│  ├── 国学大师数据转换                                                    │
│  ├── 维基文库精选导入                                                    │
│  └── 用户反馈驱动的内容扩充                                              │
└─────────────────────────────────────────────────────────────────────────┘
```

### 6.2 技术实施要点

| 要点 | 说明 |
|------|------|
| **EPUB 解析** | 复用现有 EPUB 解析逻辑，注意中文编码 |
| **章节识别** | 中文书籍章节标题格式多样，需适配 |
| **难度分析** | 需开发中文难度评估算法（字频、词汇等级） |
| **繁简转换** | 好读为繁体，需考虑简繁转换 |
| **元数据** | 中文书籍元数据不规范，需清洗补充 |

### 6.3 版权注意事项

| 类型 | 版权状态 | 商用风险 |
|------|----------|----------|
| 古籍 (1949年前) | 公共领域 | 低 |
| 民国作品 | 需核实作者去世时间 | 中 |
| 现代作品 | 作者去世后50年 | 高 |
| 翻译作品 | 译者去世后50年 | 高 |

**建议**: 优先导入明确为公共领域的古籍和经典文学

## 7. 与英文资源对比

| 维度 | 英文资源 (Gutenberg/SE) | 中文资源 |
|------|------------------------|----------|
| **数量** | 76,000+ / 1,000+ | 分散，总量大 |
| **格式规范** | EPUB3 标准 | 格式多样，需转换 |
| **元数据** | 丰富且结构化 | 需要补充和标准化 |
| **API** | Gutendex、OPDS | 大多无 API |
| **质量** | SE 高，PG 参差 | 好读/文硕阁较高 |
| **版权** | CC0/公共领域明确 | 需逐一核实 |
